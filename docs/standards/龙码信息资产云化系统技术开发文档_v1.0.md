# 龙码信息资产云化系统技术开发文档

版本：V1.0  
状态：云化系统技术开发母本  
定位：在《龙码信息资产本地化系统》基础上，补齐线上云端、多租户、同步协作、授权供给、统一审计与商业化交付能力。

---

## 一、文档定位

本文档不是重新设计一套信息资产系统，也不是推翻本地化系统。

本文档只做一件事：

> **把龙码信息资产本地化系统，升级为可线上运行、可多租户使用、可本地云端同步、可授权调用、可协作治理、可商业化交付的信息资产云化系统。**

### 1.1 与本地化系统的关系

本地化系统解决：

```text
个人 / 企业本地资料
→ 本地准入
→ 本地信炉
→ 本地治理
→ 本地供给
→ 本地回源与审计
```

云化系统解决：

```text
多企业 / 多项目 / 多端资料
→ 云端准入
→ 云端信炉
→ 云端治理
→ 云端供给 API
→ 云端回源与统一审计
→ 本地云端双向同步
```

### 1.2 云化系统不改变的主链

云化系统必须继承本地化系统的完整主链：

```text
吸金 / 准入
→ 龙码信炉
→ 四级知识结构
→ 元数据治理
→ 治理型向量知识库
→ 供给
→ 回源
→ 审计
→ 生命周期管理
```

### 1.3 云化系统新增的能力

相对本地化系统，云化版新增 8 类能力：

1. 多租户空间
2. 云端统一身份与授权
3. 本地云端同步
4. 企业协作治理
5. 云端对象存储
6. 云端治理型向量服务
7. 标准供给 API
8. 计量、配额与商业化交付

## 二、系统定位

### 2.1 它是什么

龙码信息资产云化系统是：

- 龙码信息资产系统的云端形态
- 龙码信息供应链系统的 SaaS / 私有云 / 混合云实现
- 面向企业 AI、Agent、数字员工、行业系统的信息资产供给云
- 本地化系统的同步中心、授权中心与协作中心

### 2.2 它不是什么

它不是：

- 普通云盘
- 普通企业知识库
- 普通 RAG 平台
- 单一向量数据库服务
- 单一聊天机器人后台
- 本地化系统的替代品

### 2.3 最准确的一句话定义

> **龙码信息资产云化系统，是在龙码协议约束下，以云端多租户、同步协作、信炉炼化、四级知识结构、治理型检索、供给 API、回源审计与生命周期管理为核心，把多来源资料转化为可治理、可授权、可调用、可回源的信息资产云。**

## 三、云化系统与本地化系统的分工

| 维度 | 本地化系统 | 云化系统 |
|---|---|---|
| 部署位置 | 本地电脑 / 企业内网 / 私有服务器 | 公有云 / 私有云 / 混合云 |
| 数据主权 | 本地优先 | 租户隔离 + 授权控制 |
| 资料入口 | 本地文件、URL、API | 多端上传、URL、API、组织协作、本地同步 |
| 存储 | 本地文件系统 + SQLite/LanceDB | OSS/S3/MinIO + PostgreSQL + 云向量库 |
| 协作 | 个人或小团队 | 企业组织、多角色、多项目 |
| 供给 | 本地 API | 标准云 API / SDK / Webhook |
| 审计 | 本地 trace | 云端统一审计、租户级审计 |
| 商业化 | 私有产品 / 本地版 | SaaS / 私有云 / 混合云授权 |

### 3.1 本地优先原则

本地化系统仍然是资产主权和私有资料入口的重要形态。

云化系统不能要求所有资料都必须上传云端。

必须支持三种模式：

1. **纯云端模式**：资料、治理、向量、供给都在云端
2. **本地优先模式**：原始资料留本地，云端只保存摘要、锚点、索引或授权后的资产
3. **混合模式**：冷层在本地，温层/热层部分同步到云端

## 五、核心主链：云吸金 → 云点金 → 云沉金 → 云供金 → 云回金

云化系统延续本地化系统的金流逻辑，但增加租户、权限、同步、协作与云端审计。

### 5.1 云吸金

从云端入口吸收资料原矿。

来源包括：
- 企业上传文件
- 本地系统同步
- URL / 网页 / 站点抓取
- 第三方 API
- 企业系统数据
- 会话记录
- 代码仓库材料
- 日志与运维资料
- 多媒体转写结果

输出对象：
- `cloud_raw_asset`
- `tenant_id`
- `workspace_id`
- `source_ref`
- `storage_uri`
- `trace_id`
- `ingest_policy`

### 5.2 云点金

云端龙码信炉执行：
- 抽取
- 切片
- 脱水
- 蒸发
- 去重
- 聚类
- 摘要
- 锚点提取
- 版本比对
- 协议对象生成

### 5.3 云沉金

沉淀为：
- 云端知识单元
- 摘要单元
- 证据单元
- 锚点单元
- 版本镜像单元
- 上下文包索引
- 同步状态记录

### 5.4 云供金

向上层系统提供：
- 检索包
- 推理包
- 执行包
- 回源包
- 审计包
- Agent 可调用上下文包

### 5.5 云回金

当证据不足、冲突过高、上下文不充分、权限不足时：
- 回源云端原文
- 回源网页快照
- 回源本地资产引用
- 回源版本镜像
- 触发本地补资料请求
- 写入云端审计

## 模块 3：本地云端同步服务

### 定位
连接本地化系统与云化系统。

### 同步模式

#### A. 全量同步
本地原文、元数据、节点、向量索引摘要全部同步到云端。

#### B. 元数据同步
原始文件留本地，只同步：
- 文档记录
- 协议节点
- 摘要
- 锚点
- source_ref
- storage_pointer

#### C. 索引同步
只同步热层必要字段：
- 向量引用
- chunk 摘要
- scope
- mass_score
- evidence_anchor

#### D. 回源代理模式
云端只保存索引，本地作为回源代理。

### 同步核心字段
- `sync_id`
- `local_asset_id`
- `cloud_asset_id`
- `sync_mode`
- `sync_status`
- `local_hash`
- `cloud_hash`
- `last_synced_at`
- `conflict_status`

### 冲突处理
- 本地优先
- 云端优先
- 新版本优先
- 人工确认
- 双版本并存

## 模块 5：四级知识结构云端治理

所有正式知识必须统一纳入：


### 云端增强要求
1. 作用域必须绑定 tenant
2. workspace 可以承载多个企业或项目
3. project 必须支持跨成员协作

3. 跨租户知识默认互通

## 模块 7：云端治理型向量知识库

### 定位
不是普通向量库，而是治理后高价值候选热层。

### 检索流程
```text
请求进入
→ 租户校验
→ 权限校验
→ 作用域门控
→ Top-K 语义召回
→ 治理型精排
→ 证据锚点过滤
→ 回源条件判断
→ 输出供给包
→ 写审计
```

### 首版精排公式
```text
final_score = semantic_score
            × authority_score
            × trust_score
            × freshness_weight
            × scope_weight
            × status_weight
            × permission_weight
```

### 禁止
- 只按 cosine 返回最终结果
- 跳过权限校验
- 跨租户裸召回
- revoked / expired 默认进入供给

## 模块 8：云端供给 API

### 定位
给上层 Agent、行业系统、AI 助手提供可治理上下文。

### 供给类型
1. 检索包 RetrievalPack
2. 推理包 ReasoningPack
3. 执行包 ExecutionPack
4. 回源包 SourcePack
5. 审计包 AuditPack

### 供给原则
1. 只供给被治理后的资产
2. 必须带 source_ref
3. 必须带 evidence_anchor
4. 必须带 trace_id
5. 高风险供给必须带回源建议
6. 资料不足时返回 HOLD + required_evidence

### 回源接口必须返回

```json

  "reason": "SOURCE_REQUIRED_FROM_LOCAL",
  "required_source": {
    "local_asset_id": "RA_001",

```

## 10.2 为什么云端不直接照搬 SQLite + LanceDB

本地化首版使用 SQLite + LanceDB 是为了个人版轻量运行。

云端必须支持：
- 多租户
- 并发
- 权限
- 事务
- 审计
- 配额
- 扩展

因此云端首版建议用 PostgreSQL 作为温层治理库，向量层首版可用 pgvector 降低复杂度，后续再拆成独立向量服务。

## 十七、最终定义

龙码信息资产云化系统不是云盘，不是普通知识库，不是普通 RAG 平台。

它的正式定义是：

> **在龙码协议约束下，以多租户云端空间、本地云端同步、云端龙码信炉、四级知识结构、元数据治理、治理型向量知识库、标准供给 API、回源审计与生命周期管理为完整闭环，把多来源资料转化为可治理、可授权、可调用、可回源、可审计的信息资产云。**

一句话收束：

> **本地化系统解决资产主权与私有供给，云化系统解决资产协作、同步、授权、调用与商业化交付。**
